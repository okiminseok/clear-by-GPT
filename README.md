# CLEAR

미루는 일을 10초-30초짜리 쉬운 행동으로 쪼개서 한 번에 하나씩 보여주는 웹 앱입니다.

## 제품 방향

CLEAR는 일을 게임처럼 꾸미는 앱이 아니라, 현실의 할 일을 게임처럼 이어지게 만드는 앱입니다.

캐릭터, 포인트, 상점 같은 외적 보상보다 사용자가 실제로 움직였다는 감각을 중요하게 봅니다. 할 일을 작고 구체적인 행동으로 나누고, 하나를 끝낼 때마다 진행률과 짧은 코치 멘트로 바로 다음 행동으로 넘어가게 만듭니다.

핵심 루프는 다음과 같습니다.

1. 부담스러운 일을 입력한다.
2. 10초-30초짜리 쉬운 행동으로 쪼갠다.
3. 한 화면에 한 행동만 보여준다.
4. 사용자가 `했어`를 누르면 진행률이 바로 움직인다.
5. 완료한 일은 점수보다 현실이 바뀐 증거로 남긴다.

설계 원칙:

- 첫 단계는 말도 안 되게 쉬워야 합니다.
- 보상은 앱 안의 장식이 아니라 현실 변화에서 느껴져야 합니다.
- 사용자가 앱을 보며 다음 단계로 넘겨야 하므로 휴대폰, 화면, 브라우저, 앱을 못 쓰게 만드는 단계는 만들지 않습니다.
- 완료 기록은 사용자를 압박하는 관리표가 아니라 "내가 실제로 움직였다"는 흔적으로 보여줍니다.

## Vercel 환경변수

Anthropic API 키를 Vercel 프로젝트 환경변수에 추가하세요.

- 필수 이름: `ANTHROPIC_API_KEY`
- 선택 모델명: `ANTHROPIC_MODEL` 기본값은 `claude-haiku-4-5`

Vercel에서는 `ANTHROPIC_API_KEY`에 Anthropic API 키를 넣으면 됩니다. 기본 모델은 `claude-haiku-4-5`입니다.

환경변수를 새로 추가하거나 이름을 바꾼 뒤에는 Vercel에서 지금 테스트하는 환경을 다시 배포해야 적용됩니다. Production URL이면 Production에, Preview URL이면 Preview에도 환경변수를 켜고 재배포하세요.

앱에서 계속 API 키 오류가 뜨면 오류 문구 끝의 `감지된 키`를 확인하세요. `감지된 키: 없음`이면 현재 배포된 Vercel 서버 함수가 `ANTHROPIC_API_KEY`를 읽지 못하는 상태입니다.

## 로컬 테스트

정적 화면만 확인하려면 `index.html`을 열면 됩니다. `/api/split`까지 함께 테스트하려면 Vercel CLI 또는 Node 서버리스 환경에서 실행하세요.

```bash
vercel dev
```
